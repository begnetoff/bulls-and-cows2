Rules & Usage

Bulls and cows is a small Node.js project. Some actions (like starting the Snake mini-game) depend on the server being available.

General rules

  • Use the site normally — no spamming requests or trying to break endpoints.
  • No scripts/autoclickers to brute-force actions or flood the server.
  • If something fails, refresh the page and try again after a few seconds.

Navigation

  • Use the arrows to switch sections (it loops: 1 → 2 → 3 → 1).
  • Keyboard: ← / →

Snake: start conditions

The Play button should start Snake only when the backend responds. This prevents the game from running in an invalid state.

Expected behavior

  • If the server responds: Snake can start.
  • If the server does not respond: the site should block the action and show a message.
  • This is expected behavior, not a browser problem.

Typical messages

  • “Server is offline”
  • “No response from server”
  • “Try again later”

Availability & data

Short downtime can happen during updates, restarts, or deployments. If the server is down, some features may be temporarily disabled.

Availability expectations

  • The server might be unavailable outside “working hours”.
  • Temporary downtime is allowed during maintenance.
  • Slow network can make requests feel delayed.

Privacy (simple)

  • No personal data is required to read these rules.
  • Basic technical logs may exist (requests/errors) for debugging.

Reporting issues

  • Report bugs via the project’s issue tracker.
Design: sharp corners, simple motion, logo-driven palette.
